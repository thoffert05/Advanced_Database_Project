# CPSC 531 Project
My code for the class
Anthony Hoffert
-Common Core: from my old job, not really related to this project but used by my helper programs, this project contains basic helper functions
-Progress: from my old job, not really related to this project but used by my helper programs, this project contains a very nice progress bar that can handle multiple progress bars at the same time
-Logger: from my old job, not really related to this project but used by my helper programs, this project contains a nice GIU logger, for storing, showing and displaying Logs in a nice form
-Data_Uploader_To_Spark_On_Cloud: this program just uploads a folder with CSV files to a google cloud bucket, it was used to upload all the AIS CSV files and the cruise file to google cloud
-MMSI cruise ship mapper: this was a helper program that maps the MMSI numbers to cruise ships for the cruise ship CSV, I still had to do some manual mapping since AIS ship names do not match expected cruise ship names, and AI had to help with some
-NOAA_Downloader: this program downloaded AIS CSV zip files from the NOAA government website, it takes a few days of continous running to download a full year.
-python: this folder contains various python files that were used for ingestion into Spark from the google cloud bucket, ingestion.py is launched and it downloads all the file from google cloud bucket, it cleans the CSV files mid memory stream and writes it to local, then moves it to HDFS then calls a python file called spark_ingest.py to ingest the files from HDFS temp into parquet files into HDFS /data/ais_raw partitioned by day, momentum_table_creator is a python file that runs on yarn and calculates the momentum values and writes them as parquet files in seveal folders raw ais data for cruise ships are written to HDFS /data/momentum_raw, it also computes summaries for each ship, each cruiseline, and each day and writes them to HDFS /data/summary_momentum_daily then it also computes the summaries for each ship, each cruiseline, each year and writest that to HDFS /data/summary_momentum_yearly
in python api.py is used to interface with FastAPI to handle get requests and retrun json text it uses.
-python/web_hdfs_interface/hdfs_interface.py: this interfaces with HDFS and allows Polars to see the files on HDFS as listings, and also get the data in the parquets as a dataframe
-python/web_momentum_loader/initial_momentum.py: this reads the yearly summary written by spark sql and returns the dataframe to api.py which returns it as a json to the caller
-python/web_momentum_loaderraw_momentum_handler.py: this reads all raw momentum data for a given date and can filter it by cruise line or by ship name and returns the data as a dataframe to api.py which returns it to the caller as a json string
-python/web_momentum_loader/ship_metadate_handler.py: this reads all metadate inside the cruise table and returns a dataframe containing all cruise ship stats as a dataframe which is returned as a json string to the caller, this is not used because all momentum data also contains cruise ship metadata so the initial_momentum is used in its place
-python/web_momentum_loader/summary_momentum_handler.py: this returns all the summary data for a given date range, it can be filtered by cruiseline and by ship name, it returns it to Api.py as a dataframe and API.py returns it as a json string to the caller
-CPSC531_Database_Website-master this project is what is running on Vercel it handles the web API, Able made most of this, Anthony modified the CPSC531_Database_Website-master/src
/Dashboard.svelte rather heavily to make the graphing look right, and rely on the fastapi json responses heavily
